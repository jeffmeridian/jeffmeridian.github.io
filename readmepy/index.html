Core Site Utilities #

1. generate_urls_permaLinks.py

  • Purpose: Automates the generation of a list of all blog post URLs.
  • How it works: Recursively scans the project for Markdown (.md) files, excluding system folders like _site and node_modules. It converts file paths into Eleventy-style permalinks (e.g., folder/file.md -> /folder/file/).
  • Output: Generates post_urls.txt.

2. generate_urls-cvs.py

  • Purpose: Creates a dataset for testing the comment system.
  • How it works: Similar to the permalink generator, but instead of a simple list, it creates a CSV file populated with placeholder data (Name: "Tim", Rating: 6).
  • Output: Generates test_data.csv.

3. submit_fake_comments.py

  • Purpose: Stress-tests or populates the Google Forms-based comment system.
  • How it works: Reads the test_data.csv file, uses the Faker library to generate realistic names, and submits comments programmatically to a Google Form using HTTP requests.
  • Configuration: Uses a hardcoded Google Form ID and entry fields.

4. merge_posts.py

  • Purpose: Consolidates all website content into a single searchable or readable file.
  • How it works: Aggregates the content of every .md file in the project. It extracts the title from the YAML front matter and adds a clear header for each source file.
  • Output: Generates timestamped files like merged_content_2025-10-04_15-38-16.txt.
  • Use Case: Extremely useful for providing a "full site knowledge base" to AI models or for global search indexing.

5. migrate_images.py

  • Purpose: Complete migration of site assets from PNG to WebP format.
  • How it works:
    1. Batch Conversion: Scans the entire project for directories named assets/ and converts all .png files to .webp. It uses premium settings: quality: 40, compression_level: 6, and preset: drawing.
    2. Auto-Cleanup: Automatically deletes the original .png file once a successful .webp conversion is confirmed.
    3. Reference Updating: Scans all Markdown files (front matter and body) for image references and updates them to point to the new .webp extension.
  • Use Case: This tool is used to significantly reduce site weight and improve performance by transitioning to modern image formats.

Game-Specific Tools #

games/slopped04/update_pdf.py & games/slopped05/update_pdf.py

  • Purpose: A specialized pipeline for converting PDF "zines" into web-optimized image galleries.
  • Workflow:
    1. Extracts pages from a specified PDF using PyMuPDF (fitz).
    2. Uses ffmpeg to convert those pages into WebP format.
    3. Applies premium compression settings: quality: 40, compression_level: 6, and preset: drawing for maximum readability at minimum file size.
    4. Crucially: It then automatically overwrites the index.html file in the same directory to update the gallery with the new image count and language support.

games/maumau/images/cards/list_files.py

  • Purpose: Simple inventory tool.
  • How it works: Lists every file in the card images directory and writes it to a reference file.
  • Output: Generates file_list.txt.

games/maumau/images/cards/files.py
